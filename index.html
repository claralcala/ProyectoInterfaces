Introducción

Urban Stride es la aplicación de escritorio de la tienda del mismo nombre, realizada en Java haciendo uso de Java Swing para el diseño y la codificación de la parte de interfaz gráfica. Los datos se almacenan en una base de datos local MySQL.

El objetivo de la aplicación es hacer las veces de tienda, el usuario podrá crear su cuenta de usuario, loguearse y, posteriormente, una vez que esté implementada esa función, comprar y visualizar su historial de pedidos.

Este manual busca, en primer lugar, explicar cómo se empaqueta y ejecuta la misma. En segundo lugar, se explicará al usuario cómo se hace uso de la aplicación, así como las distintas funciones que pueden llevarse a cabo con ella y, por último, se repasará la estructura interna y cómo está estructurado el código de la aplicación.

Conexión

La clase Conexion, tal y como su nombre indica, es la encargada de establecer la conexión con la base de datos MySQL y se encuentra en el paquete “controlador”.

Lo primero que hace es crear una instancia de la clase Connection de java.sql.

Lo siguiente son los datos de conexión a la base de datos, que se declaran a través de String. Estos datos deben incluir necesariamente el driver de MySQL, el usuario de la base de datos (en nuestro caso, root), la contraseña para conectarse a la base de datos (ad1234) y la URL. Recordemos que en nuestro caso la base de datos se encuentra en local, usando el puerto por defecto de MySQL, 3306.

Lo último es una función a la que llamaremos cuando necesitemos hacer algo en la base de datos. Esta función inicializa la instancia de Conection que creamos previamente usando los datos que ya hemos declarado y, por último, devuelve esa instancia de la clase Connection (en nuestro caso, se llama “con”).

ConsultasBD

Esta clase también forma parte del paquete “conexion”. En este caso, es donde realizamos las consultas a la base de datos y también tenemos otros métodos que sirven para propósitos como encriptar o desencriptar la contraseña. Los veremos uno por uno.

En primer lugar, tenemos el método guardarUsuario.

Este método recibe por parámetro un objeto Usuario y lo que hace es guardarlo en la base de datos.

Lo primero es crear una instancia de la clase Conexion que ya explicamos previamente. Luego, y muy importante, creamos una variable String para la contraseña encriptada. La contraseña se encripta llamando a un método que posteriormente veremos.

Luego toca la sentencia SQL, que en este caso, será de insert. Nosotros usaremos un PreparedStatement porque es seguro ante ataques como la inyección SQL. Comprobaremos si el nombre de usuario ya existe porque, en nuestro caso, debe ser único. El método para esto lo explicaremos también después.

Posteriormente especificamos todos los parámetros que se insertarán, que corresponden a los atributos de nuestra clase Usuario. Ejecutamos la actualización sobre la base de datos (es decir, el insert), y avisamos del resultado al usuario de la aplicación con JOptionPane.  Lo último es cerrar la conexión.

El segundo método es consultarUsuario y lo que hace es permitir el login. Se le pasa por parámetro el usuario y la contraseña que obtendremos de los campos de texto de la aplicación.

Al igual que antes, establecemos la conexión y usamos PreparedStatement. Luego lo que hacemos es inicializar unas variables con lo que obtenemos de la base de datos, estas variables las usamos para comparar con los datos que obtenemos por parámetro. De esta forma, si el usuario introduce un username o una contraseña que no corresponden, le podemos avisar mediante JOptionPane, igual que antes. Para verificar la contraseña usamos un método específico porque necesitamos desencriptarla.

La razón por la que este método devuelve un booleano es porque lo necesitamos para cerrar la ventana de Login. Solo si es true, la ventana de Login se cerrará y aparecerá una instancia de la ventana de la tienda, para que el usuario pueda proceder a realizar un pedido.

Con el método usuarioExistente lo que hacemos es comprobar si ese nombre de usuario ya existe.

Lo que hace es simplemente establecer una consulta que cuente los usuarios con determinado username que se le pasa por parámetro. Si hay 1, es que ese nombre de usuario ya existe. Si no, devolvemos false, porque ese nombre de usuario no está en uso, así que puede registrarse sin problema.

Los últimos dos métodos sirven para encriptar y desencriptar la contraseña.

Con hashPassword usamos la clase MessageDigest y la codificación SHA-256. Esto lo que hace es convertir la contraseña en un array de bytes. Luego lo reconvertimos a String para poder insertarlo en la BD (porque en nuestra base de datos la contraseña es de tipo VARCHAR).

El método verificarPassword lo que hace es comprobar que la contraseña que se la pasa por parámetro (inputPassword) coincide con la que está almacenada en la base de datos. Lo primero es obtener el hash de la contraseña pasada por parámetro. Si coincide con el hash almacenado en la base de datos, lo devolvemos como verdadero. La contraseña es correcta.

Modelo Usuario

Es simplemente la clase modelo/objeto que usaremos para insertar usuarios en la base de datos. Contiene los atributos de la clase, el constructor vacío, los getters y setters y el método toString.

Los atributos son el id, username, contraseña, nombre, apellidos, correo electrónico, teléfono y dirección. El id se genera automáticamente de forma autoincremental en MySQL, por lo que nosotros no tenemos que preocuparnos por él.
